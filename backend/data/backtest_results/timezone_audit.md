## 3. pytz imports in production

`pytz` is deprecated since Python 3.9 — `zoneinfo.ZoneInfo` is the standard. 
`pytz` still works correctly for DST, but mixing both creates inconsistencies.

## 7. Conclusion

**Action required**: 5 HIGH-severity naive-time site(s).
Replace `datetime.now()` with an explicit ET clock for any session/daily-reset decision.
Backtest is unaffected (timestamps come from market data); paper/live deployments on UTC hosts
would fire daily resets at the wrong wall-clock moment.

Recommended fix (deferred — paper/live not yet a priority per §11 sprint order): introduce
`engines/clock.py` exposing `now_et() -> datetime` (TZ-aware in `America/New_York`); replace
the 5 HIGH sites + the `datetime.now()` fallback in `setup_engine_v2.py:89` with `now_et()`.
Backtest passes timestamps from data and never reaches these fallbacks, so no behavioral
change for current campaigns. Paper/live correctness restored when the fix lands.
